# my-story-helper

此系统目标为ai辅助写小说，人工润色

使用技术：react + nodejs + sqlite

基本规划：
用户提供大纲，可通过大纲生成章节目录及细纲
通过细纲生成本章内容，同时生成本章人物、物品、势力等向量
生成章节内容时通过选取参与角色、势力等内容检索向量，将其与细纲、大纲等打包发给ai，然后生成章节内容

升级：增加多agent角色，比如检查错别字的、检查是否当前出现了已死亡角色或已损毁武器的这些智能体，
多个大模型配置，可以给不同的角色配不同的大模型


当前实现：
对接deepseek 和 text-embedding-v3
前者实现根据大纲生成章节目录，根据细纲生成文章内容，根据文章内容生成摘要
后者实现根据摘要生成向量
在生成文章时，会检索向量，如果命中，就会和大纲、卷细纲、前后五章细纲 一起组成提示词


> AI 驱动的长篇创作平台。

## 技术栈

| 层 | 技术 |
|---|---|
| 后端框架 | nodejs + express |
| AI 模型 | deepseek + text-embedding-v3 |
| 向量数据库 | sqlite |
| 主数据库 | SQLite |
| 前端 | react + TypeScript + Vite + antdesign |
| 状态管理 | zustand |
